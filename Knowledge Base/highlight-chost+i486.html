Wysłany: Czw Sie 30, 2007 7:05 pm    Temat postu:
This happens a lot. Stray files are probably left over from the tarball compiler as well as the profile being screwy. You should edit /etc/env.d/05gcc and /etc/env.d/gcc/* to correct the paths, then re-run gcc-config and select the correct profile. If any files in /etc/env.d/gcc are named i486-pc-linux-gnu-* get rid of them first, then run gcc-config. Just to be sure, check the output of this:
Kod:

emerge -Ovp gcc


If an older version is showing up in another slot, then remove it like so:
Kod:

emerge -C =gcc-<old version>


Now, check /usr for a directory named i486-pc-linux-gnu. You should only have i686-pc-linux-gnu, so if the i486 one exists, nuke it. Next check if the directory /usr/libexec/i486-pc-linux-gnu exists, and again remove it if it does. Then see if /usr/lib/gcc/i486-pc-linux-gnu exists. Kill that too if you find it. Now do this:

Kod:

rm -f /usr/bin/i486-pc-linux-gnu-*


If the i486 compiler was an older version, also check /usr/bin for any files matching the pattern *-<old version> and get rid of those too. Lastly, look for and, if it exists, get rid of /usr/share/gcc-data/i486-pc-linux-gnu/

You should also check through this doc:

Changing the CHOST variable

as that's effectively what has been done, intentional or not.
